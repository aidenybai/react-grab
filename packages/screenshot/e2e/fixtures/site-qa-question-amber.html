Why does drawing an SVG blob URL onto canvas taint it in Chromium?

Asked 2 years ago · Modified 3 months ago · Viewed 48k times
▲127▼

I serialize a DOM subtree into an SVG with foreignObject and draw it on a canvas. With a data URL everything works, but with a blob URL the canvas becomes tainted and toDataURL() throws a SecurityError: 

const url = URL.createObjectURL(svgBlob);
img.src = url; // canvas tainted after drawImage
ctx.drawImage(img, 0, 0);
canvas.toDataURL(); // SecurityError

Why are blob URLs treated differently from data URLs here?

This is tracked in whatwg/html#10641. Chromium treats SVG-in-blob as potentially able to reference other same-origin resources whose loads it cannot prove side-effect free, so it conservatively taints. Data URLs carry the entire document inline, so the load is provably self-contained — keep everything on data:image/svg+xml and the canvas stays origin-clean in all engines.
