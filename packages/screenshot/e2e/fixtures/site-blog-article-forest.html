The browser is the best screenshot engine you already have

Why re-implementing CSS in JavaScript was always a dead end

For a decade, taking a screenshot of a DOM node meant one thing: html2canvas. It parsed your styles, walked your layout, and painted every border-radius by hand onto a canvas. It was heroic. It was also doomed.

Every line of CSS the platform ships is a line your library now has to reimplement — forever.

The alternative hides in an SVG element from 2001. Wrap your markup in a foreignObject, hand it to the browser as an image, and the real layout engine does the painting: 

const svg = `<svg xmlns="..."><foreignObject>${markup}</foreignObject></svg>`;
img.src = "data:image/svg+xml," + encode(svg);
ctx.drawImage(img, 0, 0);

The hard part is not the rasterization — it is capturing the state the browser will not serialize for you: computed styles, form values, canvas frames, and fonts locked behind cross-origin stylesheets.
